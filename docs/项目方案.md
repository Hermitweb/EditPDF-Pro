## 1. 项目定位

### 1.1 产品定位

一款**一站式PDF桌面编辑器**，覆盖日常办公中所有PDF操作需求：

- **文字处理**：查找替换PDF中的姓名、金额、日期等关键信息
- **页面管理**：合并、拆分、插入、删除、旋转页面
- **内容标注**：添加文字/图片水印、注释高亮
- **文件处理**：压缩、加密解密、格式转换（扩展方向）

**核心理念**：用户不需要为了不同操作打开不同软件。

### 1.2 目标用户画像

| 维度 | 描述 |
|:----|:-----|
| 用户 | 个人办公人员，处理合同/协议/报表PDF |
| 操作频率 | 日常使用，每周多次 |
| 技术水平 | 普通办公用户，期望操作直观 |
| 设备 | i3-4170 / 8GB RAM / 120GB SSD |
| 系统 | Windows 7 / 10 / 11 |

### 1.3 设计目标

| 维度 | 目标 |
|:----|:----|
| **视觉** | 采用 Fluent Design 设计语言，与Win11原生风格一致 |
| **体验** | 操作路径短，核心功能2次点击内可达 |
| **性能** | 低配机流畅运行，启动≤3秒，200页PDF不卡 |
| **功能** | 一站式覆盖90%日常PDF操作需求 |
| **交付** | 单文件exe，绿色免安装，< 50MB |

### 2.2 为什么是 PyMuPDF？

| 需求 | PyMuPDF | PDFBox (Java) | iText 7 |
|:----|:-------:|:-------------:|:-------:|
| 查找已有文字 | ✅ `search_for()` 一键定位 | ❌ 需自建内容解析器 | ✅ 需PdfCanvasProcessor |
| 替换已有文字 | ✅ 遮盖+插入，~50行搞定 | ❌ 极难，几百行 | ✅ 中等 |
| 合并/拆分 | ✅ 原生API | ✅ | ✅ |
| 中文支持 | ✅ 好 | ✅ 好 | ✅ |
| 性能(C底层) | ⚡ 极快 | 🐢 JVM较重 | 🐢 JVM较重 |
| 内存占用(100页) | ~80MB | ~300MB+ | ~250MB |
| 许可证 | AGPL (个人免) | Apache 2.0 | AGPL |

**对低配机（i3-4170, 8GB）来说，PyMuPDF的C底层性能优势非常关键。**

### 2.3 为什么是 QFluentWidgets？

**QFluentWidgets**（[GitHub: zhiyiYo/PyQt-Fluent-Widgets](https://github.com/zhiyiYo/PyQt-Fluent-Widgets)）是一个国产开源项目，在PyQt5上实现了完整的 **Microsoft Fluent Design** 组件库。

| 对比 | 原始 PyQt5 | QFluentWidgets |
|:----|:---------:|:--------------:|
| 视觉效果 | Qt95年代味 | Win11 Fluent Design |
| 导航方式 | 传统菜单栏 | Fluent Navigation界面 |
| 明暗主题 | 需手写QSS | 内置深色/浅色主题 |
| 窗口效果 | 无 | 亚克力/云母特效 |
| 动画 | 无 | 平滑过渡动画 |
| 组件丰富度 | 基础 | 现代化组件 |
| Win7兼容 | ✅ | ✅ PyQt5版本 |
| 打包体积增量 | 基准 | +2MB |

**效果预览**（文字描述）：

### 4.3 关键UI设计要点

#### 导航面板（左侧）
- **FluentNavigationPanel** 组件
- 每个导航项 = 图标 + 文字
- 当前选中项高亮（主色背景）
- 收起/展开切换（窄模式只显示图标）
- 分组：核心操作 / 页面管理 / 标注工具 / 设置

#### PDF预览区（中央）
- 两栏模式：
  - **单页模式**：一页居中显示
  - **连续模式**：所有页垂直排列（默认）
- 缩放手动：50% / 75% / 100% / 150% / 200%
- 匹配宽度：自动缩放适配预览区宽度
- 匹配页面：完整显示一页
- 键盘翻页：↑↓ / PageUp PageDown / 滚轮

#### 底部工具栏
- 缩放控制（下拉+滑块）
- 页面导航（页码输入框 + 上一页/下一页按钮）
- 当前文件信息

#### 查找替换面板（底部可折叠面板）
- 搜索输入框（带清除按钮）
- 替换输入框
- 高亮匹配项计数 "共找到 N 处"
- 操作按钮组：上一个 / 下一个 / 替换 / 全部替换
- 匹配列表（点击跳转到对应位置）
- **可折叠**：点击标签展开/收起

#### 对话框
全部使用 QFluentWidgets 提供的 **FluentDialog** 风格：
- 合并对话框：多文件列表（可拖拽排序）+ 合并按钮
- 拆分对话框：范围选择器 + 预览
- 水印对话框：实时预览水印效果

#### PDFEngine (core/pdf_engine.py)

```python
class PDFEngine(QObject):
    """PDF引擎 — 整个应用唯一操作PDF的入口"""

    # 信号
    document_loaded = Signal(str, int)     # 文件路径, 页数
    page_changed = Signal(int, int)        # 当前页, 总页数
    operation_completed = Signal(str)      # 操作名称
    error_occurred = Signal(str)           # 错误信息

    def open(self, filepath: str) -> bool
    def save(self, output_path: str) -> bool
    def save_as(self, output_path: str) -> bool
    def close(self)

    # 文档信息
    def page_count(self) -> int
    def current_page(self) -> int
    def page_size(self, page_num: int) -> tuple[float, float]
    def document_info(self) -> dict

    # 导航
    def go_to_page(self, page_num: int)
    def next_page(self)
    def prev_page(self)

    # 渲染
    def render_page(self, page_num: int, dpi: int = 120) -> QImage
    def render_thumbnail(self, page_num: int) -> QPixmap

    # 文字操作 → 委托给 TextOps
    def find_text(self, text: str, case_sensitive: bool = False) -> list[FindResult]
    def replace_text(self, result_id: int, new_text: str) -> bool
    def replace_all(self, new_text: str) -> int  # 返回替换数量

    # 页面操作 → 委托给 PageOps
    def merge_pdfs(self, filepaths: list[str]) -> bool
    def split_pdf(self, page_ranges: list[tuple[int, int]]) -> bool
    def insert_pages(self, filepath: str, after_page: int) -> bool
    def insert_blank_page(self, after_page: int, size: str = "A4") -> bool
    def delete_pages(self, page_nums: list[int]) -> bool
    def rotate_pages(self, page_nums: list[int], angle: int = 90) -> bool
    def reorder_pages(self, new_order: list[int]) -> bool

    # 水印操作 → 委托给 WatermarkOps
    def add_text_watermark(self, text: str, config: WatermarkConfig) -> bool
    def add_image_watermark(self, image_path: str, config: WatermarkConfig) -> bool

    # 注释操作 → 委托给 AnnotationOps
    def add_highlight(self, page_num: int, rect: tuple) -> bool
    def add_text_annotation(self, page_num: int, text: str, pos: tuple) -> bool
```

### 8.1 渲染性能（最关键）

```python
# 三级渲染策略
THUMBNAIL_DPI = 40      # 缩略图 (A4约331×468px)
PREVIEW_DPI = 120       # 预览渲染 (A4约993×1403px)
EXPORT_DPI = 200        # 导出/打印

# 三页缓存 (当前页 + 前后各1页)
class PreviewCache:
    """只缓存可见区域附近的页面，其余不加载"""
    def __init__(self):
        self._cache: dict[int, QImage] = {}
        self._max_size = 3

    def get(self, page_num: int, render_func) -> QImage:
        # 先检查缓存
        if page_num in self._cache:
            return self._cache[page_num]
        # 淘汰策略: 移除离当前页最远的
        if len(self._cache) >= self._max_size:
            farthest = max(self._cache.keys(),
                          key=lambda p: abs(p - page_num))
            del self._cache[farthest]
        # 渲染新页
        img = render_func(page_num)
        self._cache[page_num] = img
        return img
```

### 8.2 异步操作

# 所有重型操作走 QThreadPool，不阻塞UI
def on_find_clicked(self):
    text = self.find_input.text()
    # 查找在后台线程执行
    QThreadPool.globalInstance().start(lambda: self._do_find(text))

def _do_find(self, text):
    results = self.engine.find_text(text)
    # 结果用信号回传UI线程
    self.find_completed.emit(results)

## 11. 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|:----|:---:|:---:|:----|
| **中文PDF字体不匹配** | 🔴高 | 替换后文字显示为方块/乱码 | 内置思源黑体(Source Han Sans)，替换时用嵌入字体 |
| **PDF扫描件(无文字层)** | 🟡中 | `search_for()`找不到任何文字 | 检测到无文字时提示"此PDF为扫描件，建议先OCR" |
| **加密PDF** | 🟡中 | 无法打开 | 提示输入密码，PyMuPDF支持密码解密 |
| **Win7兼容** | 🟡中 | 某些API/PyQt版本不兼容 | Python 3.8 + PyQt5 5.15.4 + 在Win7上实测 |
| **大PDF(500页+)** | 🟢低 | 缩略图生成慢 | 异步批量生成 + 进度条 + 限并发 |
| **AGPL许可证** | 🟢低 | 仅个人使用无影响 | 个人使用不触发AGPL分发条款 |
| **Trae AI代码质量** | 🟡中 | 核心逻辑可能出错 | 每段核心代码写单元测试 + 手动验证 |
